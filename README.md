## 2. Перечень используемых инструментов с обоснованием выбора:

* `Selenide` — Java-библиотека с открытым исходным кодом для автоматизации веб-интерфейсов, построенная поверх Selenium WebDriver. Она упрощает написание, чтение и поддержку UI-тестов.
* `JUnit5` - самый популярный фреймворк для написания и запуска модульных (unit) и интеграционных тестов на языке Java. Он позволяет разработчикам и инженерам по тестированию проверять корректность работы отдельных частей кода (методов, классов, компонентов) автоматически и воспроизводимо.
* `Java` - один из самых популярных языков программирования для автоматизации тестирования веб-приложений. 
* `Maven` или `Gradle` - инструменты автоматизации сборки и управления зависимостями, которые сами выкачивают нужные библиотеки и их зависимости.
* `Git` -  для контроля версий кода. Помогает отслеживать изменения в коде, возвращаться к старым версиям и работать над проектом вместе с другими коллегами.
* `GitHub` - cамая популярная в мире площадка для облачного хранения кода и совместной командной разработки.
* `Intellij IDEA` - интегрированная среда разработки программного обеспечения для многих языков программирования, в частности Java. Поддержка множества языков, Умное автодополнение и анализ кода, Встроенная поддержка фреймворков (JUnit, Maven, Gradle, Selenide, Selenium).
* `Docker` - платформа для разработки, доставки и запуска приложений в контейнерах. Контейнеры — это легковесные, изолированные среды, которые содержат всё необходимое для запуска приложения (код, зависимости, библиотеки, конфигурации).
* `Postman` - программа для тестирования API,  упрощающает процесс отладки и тестирования кода.  
* `Java Faker` - библиотека для генерации фейковых (тестовых) данных в Java-приложениях. Она позволяет создавать реалистичные, но вымышленные данные, такие как имена, адреса, email, номера телефонов, даты, банковские реквизиты и многое другое. 
* `REST-assured` - библиотека для тестирования REST API на Java. Она позволяет отправлять HTTP-запросы (GET, POST, PUT, DELETE).
* `Page Object` Каждая веб-страница (или её часть) представляется как отдельный класс, где элементы страницы — поля, а действия — методы. 
* `Allure` - фреймворк для отчетов о тестировании.

2. Отсутствие доступа к тестовой среде.
3. Блокировка IP из-за частых запросов в короткий промежуток времени.
4. Автоматизация может быть затратной по времени и ресурсам, особенно если сценарии сложные.
